# Task 021: `codemap init` CLI command (v0.2.0)

**Status:** in-progress (D1, D2, D3, D4 default-approved per task-020 retro)
**Phase:** M3 / Sprint 3a → v0.2.0
**Estimate:** 1-2 hours
**Depends on:** task-020 (M3a retro identifying F1)
**Blocks:** trial v5 isolation retest (only meaningful once `codemap init` exists as the alternative install path)

## Goal

After this task: a user installing codemap-mcp for the first time runs `codemap init` in their project root and gets a ready-to-go AGENTS.md (and optionally CLAUDE.md) containing the full lifecycle policy. Solves M3a finding F1 (Codex Desktop drops MCP `server.instructions`).

## Context

M3a confirmed Codex Desktop ignores the MCP `server.instructions` field. The voice2work trial worked only after a hand-written voice2work/AGENTS.md was added. Shipping that file as a CLI-generated artifact lets every new install get the same lifecycle policy without us telling each user to copy-paste from the README.

## Decisions (defaults; override at implementation time only if a clear reason emerges)

### D1 — Which preamble files does `codemap init` create by default?

- **A.** AGENTS.md only (Codex / Codex CLI / Cursor convention)
- **B.** AGENTS.md + CLAUDE.md
- **C.** Configurable via flags

**Default: A**, with `--claude` flag adding CLAUDE.md and `--all` adding all known preamble filenames (currently AGENTS.md + CLAUDE.md). Reason: AGENTS.md is the most universal name; opt-in for client-specific files keeps the default install non-noisy.

### D2 — Overwrite behavior when target file exists?

- **A.** Always abort
- **B.** Skip with warning; `--force` overrides
- **C.** Always overwrite

**Default: B.** Predictable + non-destructive; matches `git init`-style behavior.

### D3 — Source of the preamble text?

- **A.** Static template literal in `src/cli/init.ts`
- **B.** Reuse `SERVER_INSTRUCTIONS` from `src/instructions.ts` (single source of truth)
- **C.** Generate from a template + the SERVER_INSTRUCTIONS body

**Default: C.** The MCP `instructions` field is short (180 words, one block of bullet points). The AGENTS.md file should wrap it with a "this file was generated by `codemap init`" header + a "why it exists" footer + a regenerate command. Body of the lifecycle steps comes from `SERVER_INSTRUCTIONS` so we never drift between protocol-level instructions and human-readable preamble.

### D4 — Should `codemap init` also create `.codemap/` directory?

- **A.** Yes (with `.gitkeep`)
- **B.** No, let the first MCP write create it on demand
- **C.** Yes, but only with `--seed` flag

**Default: B.** Avoid creating the directory if the user might never connect their MCP client. The directory creates itself on first agent emission.

## Deliverables

- `src/cli/init.ts` — pure function `init(opts: InitFlags, globals: GlobalOptions): Promise<CommandResult>`. Returns `{ exitCode, stdout, stderr }` like the other CLI subcommands.
- `bin/codemap.ts` — wire `init [options]` subcommand with commander.
- `src/instructions.ts` — export a new `agentsMdContent(projectName?: string)` helper that returns the full AGENTS.md body wrapping `SERVER_INSTRUCTIONS`.
- `test/unit/cli.test.ts` — new tests:
  - creates AGENTS.md with expected content shape (header + lifecycle + footer)
  - skips (warning) when AGENTS.md exists
  - `--force` overwrites when AGENTS.md exists
  - `--claude` adds CLAUDE.md
  - `--all` adds both
  - exit code 0 on success, 1 on skip-with-warning (so CI scripts can detect)
- `README.md` — new install step ("After install: `cd your-project && codemap init`") + link.
- `package.json` + `bin/codemap.ts` + `bin/codemap-mcp.ts` — bump to `0.2.0`.

## Steps

## Exit criteria

- [ ] `codemap init` in an empty dir creates AGENTS.md.
- [ ] `codemap init` again warns + exits 1, AGENTS.md unchanged.
- [ ] `codemap init --force` overwrites.
- [ ] `codemap init --claude` adds CLAUDE.md.
- [ ] `codemap init --all` adds both.
- [ ] Generated AGENTS.md contains the full SERVER_INSTRUCTIONS body verbatim.
- [ ] All tests pass; smoke test passes; publish dry-run clean.
